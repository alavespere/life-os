## How This Works

Every time a new research article or intelligence brief lands in `Project_Oracle_Feeds/`, Claude runs the **Ingest Operation**:
1. Summarize the article
2. Identify which existing vault nodes it connects to
3. Add or update the relevant entry in this index
4. Link to the full note

This index is the compounding knowledge graph. Search it before starting any research brief or content piece.

---

## Ingest Operation (run when new Oracle feed arrives)

```
/ingest [filename or topic]
```

Claude will:
1. Read the new file in Project_Oracle_Feeds/
2. Extract: key data points, source, date, risk implication
3. Search this index for related existing entries
4. Add new entry OR update existing entry with the new data point
5. Update the "Last updated" timestamp above
